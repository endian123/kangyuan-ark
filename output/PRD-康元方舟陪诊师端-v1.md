# PRD · 康元方舟陪诊师端 v1

## 📌 1. 需求背景

- **业务背景**：康元方舟是专业陪诊服务平台，需要为陪诊师提供移动端工具，用于接单、服务记录、工单回填和个人信息管理。
- **现状与痛点**：
  - 痛点 1：陪诊师缺乏统一的订单管理工具，沟通效率低
  - 痛点 2：服务过程缺乏标准化记录，服务质量难以追踪
  - 痛点 3：就诊凭证管理分散，无法形成完整服务档案

---

## 🎯 2. 目标用户 & 场景

- **主要用户**：已入驻康元方舟平台的陪诊师，年龄 25-50 岁，女性占比 70%，使用频率为日均 2-5 单
- **次要用户**：平台运营人员，用于审核入驻、派单、服务质量监控
- **核心场景**：
  1. **接单服务** — 陪诊师收到派单通知，查看订单详情，开始服务
  2. **服务过程** — 陪诊师陪同就诊，记录服务过程，上传就诊凭证
  3. **服务完成** — 陪诊师结束服务，查看服务统计数据
  4. **个人管理** — 陪诊师维护个人信息、设置服务时间和费用

---

## 👥 3. 用户故事

- **#1** As a **陪诊师**, I want **快速查看待服务订单详情**, so that **提前做好服务准备**
- **#2** As a **陪诊师**, I want **一键发送短信通知就诊人**, so that **高效沟通预约信息**
- **#3** As a **陪诊师**, I want **记录服务过程并上传凭证**, so that **形成完整服务档案**
- **#4** As a **陪诊师**, I want **查看服务统计数据**, so that **了解个人服务表现**
- **#5** As a **陪诊师**, I want **灵活设置服务时间和费用**, so that **合理安排工作节奏**

## 🧩 5. 功能描述

### F1 · 待服务订单列表 [P0]

> 展示已分配但未开始的订单，支持查看详情、联系就诊人

#### 📝 用户故事
As a 陪诊师, I want 快速查看待服务订单列表, so that 合理安排服务顺序

#### ✅ 验收标准（AC）
- [ ] AC1: 列表展示订单基本信息（就诊人姓名、医院、科室、预约时间）
- [ ] AC2: 点击订单卡片进入订单详情页
- [ ] AC3: 空态显示插图 + 引导文案「暂无待服务订单，请等待平台派单」
- [ ] AC4: 支持按预约时间排序（最近优先）

#### 🔀 交互说明（4 态）
- **正常态**：展示订单卡片列表，包含就诊人、医院、时间信息
- **空态**：显示空状态插图 + 引导文案「暂无待服务订单，请等待平台派单」
- **异常态**：网络错误显示刷新按钮，点击重新加载
- **权限**：仅已入驻陪诊师可见，未入驻引导入驻流程

#### 🔄 关键流程
1. 进入「服务」页面，默认展示「待服务」Tab
2. 查看订单列表，按预约时间排序
3. 点击订单卡片进入详情

#### 📐 数据边界
- 单次加载 20 条，下拉加载更多
- 订单状态：待服务

#### 📊 埋点
| 事件名 | 触发时机 | 参数 |
|---|---|---|
| `order_list_view` | 进入待服务列表 | `{tab: 'wait'}` |
| `order_card_click` | 点击订单卡片 | `{order_id: 'xxx'}` |

### F2 · 订单详情 [P0]

> 展示订单完整信息，提供开始服务、联系就诊人入口

#### 📝 用户故事
As a 陪诊师, I want 查看订单完整信息, so that 提前做好服务准备

#### ✅ 验收标准（AC）
- [ ] AC1: 展示服务信息（医院、科室、预约时间、服务类型）
- [ ] AC2: 展示就诊人信息（姓名、性别、年龄、联系电话）
- [ ] AC3: 提供「开始服务」按钮

1. 从订单列表点击进入
2. 查看服务信息、就诊人信息
3. 点击「开始服务」进入服务中状态
4. 或点击「发送短信」「电话联系」沟通

#### 📐 数据边界
- 服务类型：全程陪诊、取药陪诊、检查陪诊

| 事件名 | 触发时机 | 参数 |
|---|---|---|
| `order_detail_view` | 进入订单详情 | `{order_id: 'xxx'}` |
