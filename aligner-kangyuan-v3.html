📌 §1 需求背景 ▾

1.1 业务背景

康元方舟陪诊平台是一款专注于医疗陪诊服务的 O2O 平台，连接有陪诊需求的用户（C 端）与专业陪诊服务人员（陪诊师端）。陪诊师端小程序作为平台的核心服务执行端，承担着登录注册、入驻审核、接单管理、服务执行、服务记录、个人信息管理等关键职能，是整个平台服务履约链路的「最后一公里」执行终端。

中国老龄化加速、异地就医增多、医疗流程复杂（挂号-候诊-就诊-缴费-取药），导致大量患者（尤其是老年人、孕妇、异地患者、独自就医人群）对陪诊服务存在强烈需求。然而传统陪诊市场存在以下结构性问题：

  • 服务过程不可视：陪诊师是否到岗、服务时长是否达标、服务过程是否规范，平台和用户均无法实时掌握
  • 资质审核不标准：陪诊师资质（学历、执业证书）缺乏统一审核机制，服务质量参差不齐
  • 纠纷无凭证：服务过程中无结构化记录和凭证留存，发生纠纷时无法举证
  • 接单效率低：陪诊师在多个微信群、接单工具间切换，信息分散、容易遗漏
  • 服务时间不可控：陪诊师可服务时间不透明，平台无法精准派单

陪诊师端小程序正是为解决上述问题而生，通过标准化入驻流程、GPS 签到校验、服务过程结构化记录、凭证上传留存、可服务时间管理等核心能力，将陪诊服务从「黑盒」变为「白盒」，实现服务全链路可追溯。

1.3 产品定位

康元方舟陪诊师端小程序定位为陪诊师的一站式移动工作台，覆盖从入驻到接单、从服务执行到服务记录的全链路操作。核心价值主张：

  • 对陪诊师：一个 App 完成接单、签到、记录、凭证上传、结算，告别多工具切换
  • 对平台：全链路数字化管控，服务过程可追溯、资质审核标准化、派单精准化
  • 对 C 端用户：服务过程透明可视，有凭证可查，提升信任度

1.4 北极星指标

月活陪诊师人均完成服务订单数（MAU 服务订单数）

该指标综合反映了陪诊师端的活跃度、接单转化率和履约效率，是陪诊师端产品价值的直接体现。目标：上线 3 个月内 MAU 人均完成服务订单数 ≥ 15 单/月。

2.3 核心使用场景

场景 1：陪诊师首次注册并入驻

陪诊师张女士（32 岁，前护士）经朋友推荐了解到康元方舟平台，扫码打开小程序。看到微信一键登录按钮，勾选协议后一键登录。系统检测到她尚未入驻，引导她进入入驻流程：填写基本信息（姓名、手机号、身份证照片 OCR 自动识别填充）、上传护士执业资格证和学历证书、勾选协议并提交申请。3 天后收到审核通过通知，在线签署电子合同后正式成为入驻陪诊师。

关键触点：F1 微信登录 → F17 我的首页（入驻流程引导）→ F18 基本信息填写 → F20 OCR 识别 → F19 证书管理 → 提交审核 → F23 电子合同签署

场景 2：陪诊师接收并执行一次完整服务

陪诊师李先生（38 岁）早晨 8 点打开小程序，在服务页看到一条 9:30 在市第一医院的待服务订单。点击查看详情，了解就诊人信息（张大爷，72 岁，心内科就诊）和联系方式。9:00 到达医院附近，点击「开始服务」，系统进行 GPS 定位校验（确认在 1000 米范围内），拍照签到后服务状态变更为「服务中」。服务过程中记录关键节点（挂号、候诊、就诊、缴费、取药），并上传就诊凭证照片。11:00 服务结束，点击「服务完成」，系统弹窗确认（服务时长 2 小时 > 15 分钟无需二次确认），自动提交记录并发送完成短信。

关键触点：F5 订单列表 → F6 订单详情 → F7 GPS 签到 → F8 发送短信 → F10 服务记录 → F11 凭证上传 → F12 服务完成 → F14 服务总结

场景 3：陪诊师管理可服务时间

陪诊师王女士（28 岁）每周一三五有其他工作，需要设置可服务时间为周二、周四全天及周末。进入「我的」→「服务时间」，在上方星期芯片中取消周一、三、五的选中，选中二、四、六、日。然后在月历中查看具体日期安排，点击某个日期弹出编辑面板，将上午/下午圆点切换为绿色（可服务）或灰色（不可服务）。保存后立即生效，平台将根据她的可服务时间精准派单。

关键触点：F17 我的首页 → F22 可服务时间设置

场景 4：服务中遇到异常情况报备

陪诊师赵先生（40 岁）在陪诊过程中，就诊人突然需要做额外检查导致服务时间超出预期。他点击「异常报备」按钮，选择异常类型（「服务时长超出」），填写描述（「就诊人需临时增加 CT 检查，预计延长 1 小时」），拍照上传检查单后提交报备。平台运营人员收到报备通知，调整订单预期时间。

关键触点：F16 异常报备

场景 5：陪诊师查看统计数据与历史订单

陪诊师刘女士（35 岁）月末查看自己的服务数据。在服务页「服务完成」Tab 中查看统计卡片（本月接单 18 单、累计服务 156 单、进行中 1 单、评分 4.8、评价数 142 条），下拉刷新已完成订单列表，点击某条历史订单查看详情。

关键触点：F5 订单列表（服务完成 Tab）→ F13 统计与列表 → F14 服务总结

场景 6：陪诊师修改个人信息

陪诊师陈先生搬家到新城市，进入「我的」→「基本信息维护」，修改城市为「杭州市」，同时更新可服务时间和偏好医院。他尝试修改姓名但发现不可修改（需联系客服）。
